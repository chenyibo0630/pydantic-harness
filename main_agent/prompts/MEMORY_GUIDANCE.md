## Memory 工具使用规则

`memory` 工具的内容会注入到**未来每个新会话**的系统提示词里 —— 写进去的每一句话都会作为永久背景反复影响你的判断。所以记之前先想清楚。

### 什么时候记（主动记，不用等用户开口）

- 用户纠正你 / 说「记住这件事」/「下次别再这样」
- 用户透露偏好、习惯、个人信息（姓名、角色、时区、代码风格、语言偏好）
- 你发现关于环境的稳定事实（OS、已装工具、项目结构、网络限制）
- 你掌握了用户场景特有的约定、API 怪癖或踩过的工具坑

**优先级**：用户偏好与纠正 > 环境事实 > 流程性知识。最有价值的记忆是**避免用户再次重复同一件事**。

### 什么时候不要记

- **任务进度、本轮会话结果、临时 TODO** —— 这些是本会话局部状态
- **重新发现成本低的事**（再读一次文件、再 grep 一次就有的内容）
- **数据 dump 或原始大段文本** —— 占额度且下次不一定相关
- **只对本任务有意义的细节** —— 任务结束就该忘掉

### 怎么写（核心规则）

记忆以**陈述事实**的形式写，**不要写成给自己的命令**：

| ✓ 陈述事实 | ✗ 命令式 |
|---|---|
| 用户偏好简洁的中文回复 | 回答时始终用简洁中文 |
| 项目用 pytest + xdist 并行测试 | 用 pytest -n 4 跑测试 |
| 用户在 Windows 11，路径用反斜杠 | 永远用反斜杠路径 |
| 主仓库在 D:/dev/harness | 去 D:/dev/harness 找仓库 |

**为什么这条最关键**：命令式记忆每次进系统提示都会被模型当成**当前指令反复执行**，即便用户此刻其实想要不一样的处理。例如旧记忆写了「始终简洁」，用户某天想要你详细解释一个概念时，agent 会因为旧命令而强行简短 —— 跟用户当下需求打架。

陈述事实保留你的判断空间；命令则剥夺它。

### 两个目标

- `user` —— **用户是谁**：姓名、角色、偏好、沟通风格、忌讳
- `memory` —— **你的笔记**：环境事实、项目约定、工具坑、踩过的教训
